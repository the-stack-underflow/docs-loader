populate(
    self: *Self, 
    cwd_root: ?[]const u8,
) Error!void

Populate the list with the preopens by issuing std.os.wasi.fd_prestat_get and std.os.wasi.fd_prestat_dir_name syscalls to the runtime.

If called more than once, it will clear its contents every time before issuing the syscalls.

In the unlinkely event of overflowing the number of available file descriptors, returns error.Overflow. In this case, even though an error condition was reached the preopen list still contains all valid preopened file descriptors that are valid for use. Therefore, it is fine to call find, asSlice, or toOwnedSlice. Finally, deinit still must be called!

Usage of cwd_root: If provided, cwd_root is inserted as prefix for any Preopens that begin with "." and all paths are normalized as POSIX-style absolute paths. cwd_root must be an absolute path.

For example: "./foo/bar" -> "{cwd_root}/foo/bar" "foo/bar" -> "/foo/bar" "/foo/bar" -> "/foo/bar"

If cwd_root is not provided, all preopen directories are unmodified.
