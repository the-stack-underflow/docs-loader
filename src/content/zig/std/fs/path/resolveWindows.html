resolveWindows(
    allocator: Allocator, 
    paths: []const []const u8,
) ![]u8

This function is like a series of cd statements executed one after another. It resolves "." and "..". The result does not have a trailing path separator. If all paths are relative it uses the current working directory as a starting point. Each drive has its own current working directory. Path separators are canonicalized to '\\' and drives are canonicalized to capital letters. Note: all usage of this function should be audited due to the existence of symlinks. Without performing actual syscalls, resolving .. could be incorrect.
